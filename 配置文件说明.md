# 配置文件说明

本文档说明当前版本配置文件的结构、字段约束和运行时行为。项目推荐使用多规则组配置；旧单规则结构仍保留兼容，但不再作为首选写法。

## 1. 顶层结构

- 文件格式：JSON
- 默认文件名：`config.json`
- 顶层关键字段：
  - `"version": "2.0"`
  - `organization_units`

```json
{
  "version": "2.0",
  "organization_units": {
    "单位A": {
      "default": {
        "template_path": "templates/default.xlsx",
        "header_row": 1,
        "start_row": 2,
        "field_mappings": {},
        "transformations": {}
      }
    }
  }
}
```

## 2. 推荐结构：多规则组

推荐每个单位使用多规则组结构，至少包含 `default`，可按需增加 `crossbank` 或任意自定义规则组。

```json
"单位名称": {
  "template_selector": {
    "enabled": true,
    "default_bank": "中国农业银行",
    "bank_column": "开户银行",
    "special_template": "templates/crossbank.xlsx",
    "default_group_name": "农业银行",
    "special_group_name": "农行跨行"
  },
  "input_filename_routing": {
    "enabled": true,
    "routes": [
      { "project_code": "B01095", "rule_group": "b01095" }
    ]
  },
  "default": {
    "template_path": "templates/default.xlsx",
    "header_row": 1,
    "start_row": 2,
    "field_mappings": {},
    "transformations": {}
  },
  "crossbank": {
    "template_path": "templates/crossbank.xlsx",
    "header_row": 2,
    "start_row": 3,
    "field_mappings": {},
    "transformations": {}
  },
  "b01095": {
    "template_path": "templates/b01095.xlsx",
    "header_row": 1,
    "start_row": 2,
    "field_mappings": {},
    "transformations": {}
  }
}
```

运行优先级：

1. 先检查 `input_filename_routing` 是否命中
2. 若未命中，再看 `template_selector.enabled`
3. 若仍未启用，则直接使用 `default`

## 3. 兼容结构：旧单规则配置

旧写法仍可用，例如：

```json
"单位名称": {
  "template_path": "templates/default.xlsx",
  "header_row": 1,
  "start_row": 2,
  "field_mappings": {},
  "transformations": {}
}
```

限制：

- 仅用于兼容旧配置
- 不支持 `input_filename_routing`
- 新能力优先以多规则组结构维护

## 4. 规则组字段说明

以下字段适用于 `default`、`crossbank` 或其他自定义规则组。

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| `template_path` | 是 | 模板文件路径，支持相对路径 |
| `header_row` | 是 | 输出模板表头行号，从 1 开始；可为 `0` |
| `start_row` | 否 | 输出数据起始行，必须大于 `header_row` |
| `field_mappings` | 是 | 字段映射配置 |
| `transformations` | 是 | 转换规则配置 |
| `row_filter` | 否 | 输入阶段行过滤 |
| `reader_options` | 否 | 输入读取选项 |
| `clear_rows` | 否 | 输出写入前清空的数据区 |
| `fixed_values` | 否 | 模板固定值 |
| `auto_number` | 否 | 自动编号 |
| `month_type_mapping` | 否 | 用月份参数生成用途/摘要等 |
| `validation_rules` | 否 | 数据校验规则 |

约束：

- 若未显式提供 `start_row`，程序会自动设置为 `max(1, header_row + 1)`
- `header_row` 必须是整数，且 `>= 0`
- `start_row` 必须是整数，且严格大于 `header_row`
- 相对 `template_path` 会解析到程序运行目录

## 5. `field_mappings`

推荐使用完整字典格式：

```json
"field_mappings": {
  "模板字段": {
    "source_column": "输入列名",
    "target_column": "金额",
    "transform": "amount_decimal",
    "required": true
  }
}
```

字段含义：

- `source_column`：输入文件中的列名
- `target_column`：模板目标列；可选，默认等于当前映射键名
- `transform`：转换类型，默认 `none`
- `required`：字段级必填标记，供写入流程使用

`target_column` 支持以下三种写法：

- 模板表头名，例如 `"金额"`
- Excel 列字母，例如 `"A"`、`"B"`、`"AA"`
- 1 基列索引，例如 `3`

说明：

- 中文列名会优先按“表头列名”解析，不会被误判为 Excel 列字母
- 项目仍兼容旧简写格式，但建议迁移到完整字典格式

### 支持的 `transform`

- `none`
- `date_format`
- `amount_decimal`
- `card_number`

## 6. `transformations`

```json
"transformations": {
  "date_format": {
    "output_format": "YYYY-MM-DD"
  },
  "amount_decimal": {
    "decimal_places": 2,
    "rounding": "round"
  },
  "card_number": {
    "remove_formatting": true,
    "luhn_validation": true
  }
}
```

### `date_format`

- 当前仅支持输出 `YYYY-MM-DD`
- 支持的常见输入格式：
  - `YYYY-MM-DD`
  - `DD/MM/YYYY`
  - `MM/DD/YYYY`
  - `YYYY年MM月DD日`
  - `YYYY-M-D`

### `amount_decimal`

- `decimal_places`：小数位数
- `rounding` 支持：
  - `round`
  - `half_up`
  - `floor`
  - `ceil`
  - `down`
  - `up`

### `card_number`

- `remove_formatting`：去掉空格、横杠等非数字字符
- `luhn_validation`：是否启用 Luhn 校验

## 7. `reader_options`

`reader_options` 控制输入文件读取行为。

```json
"reader_options": {
  "data_only": false,
  "header_row": 1
}
```

约束：

- `data_only` 必须是布尔值
- `header_row` 必须是 `>= 1` 的整数

说明：

- `data_only=true` 时，`.xlsx` 读取公式缓存值
- `data_only=false` 时，`.xlsx` 读取公式文本
- `openpyxl` 不会计算公式，缓存值必须由 Excel 或其他兼容软件预先保存
- `reader_options.header_row` 只影响输入读取，不等同于规则组自己的 `header_row`

## 8. `row_filter`

```json
"row_filter": {
  "exclude_keywords": ["合计", "总计", "小计"]
}
```

说明：

- 当一行中任意表头值命中关键字时，该行会在读取阶段被跳过
- 常用于剔除合计行或汇总行

## 9. `clear_rows`

```json
"clear_rows": {
  "start_row": 2,
  "end_row": 200
}
```

约束：

- `clear_rows` 必须是字典
- 必须提供 `end_row` 或兼容字段 `data_end_row`
- `end_row` 与 `data_end_row` 不能同时存在
- `start_row`、`end_row` 都必须是 `>= 1` 的整数
- `start_row` 不能大于 `end_row`

行为：

- `start_row` 未提供时，默认使用规则组的 `start_row`
- `.xlsx` 在数据超出清理区间时会自动插入行
- `.xls` 要求清理区足够容纳全部数据，否则会报错

## 10. `fixed_values`

```json
"fixed_values": {
  "省份": "浙江省",
  "业务类型": "工资代发"
}
```

说明：

- 键使用模板列标识，支持列名、列字母或列索引
- 值会写入每一条输出记录

## 11. `auto_number`

```json
"auto_number": {
  "enabled": true,
  "column_name": "A",
  "start_from": 1
}
```

说明：

- `column` 与 `column_name` 都可用
- `start_from` 默认为 `1`
- 列标识同样支持列名、列字母或列索引

## 12. `month_type_mapping`

```json
"month_type_mapping": {
  "enabled": true,
  "target_column": "用途",
  "month_format": "{month}月收入",
  "bonus_value": "年终奖",
  "compensation_value": "补偿金"
}
```

说明：

- 普通处理模式下，程序会根据命令行传入的月份参数写入目标列
- `month_format` 中的 `{month}` 会替换为传入值
- 当月份参数为 `年终奖` 或 `补偿金` 时，会分别写入 `bonus_value` 和 `compensation_value`
- 合并模式下，若规则组启用该配置，会保留每行原始月类型值

## 13. `template_selector`

```json
"template_selector": {
  "enabled": true,
  "default_bank": "中国农业银行",
  "bank_column": "开户银行",
  "special_template": "templates/crossbank.xlsx",
  "default_group_name": "农业银行",
  "special_group_name": "农行跨行"
}
```

字段说明：

- `enabled`：是否启用
- `default_bank`：启用时必填
- `bank_column`：输入文件中用于判断银行的列名，默认 `"开户银行"`
- `default_group_name`：默认组输出时使用的模板名
- `special_group_name`：特殊组输出时使用的模板名
- `default_template`、`special_template`：可选；若不配置，则回退到对应规则组的 `template_path`

行为：

- `bank_column == default_bank` 时归入默认组
- 其他值归入特殊组
- 比较前会先做全角转半角和首尾空白归一化

## 14. `input_filename_routing`

```json
"input_filename_routing": {
  "enabled": true,
  "routes": [
    { "project_code": "B01095", "rule_group": "b01095" }
  ]
}
```

约束：

- 只能用于多规则组结构
- `enabled=true` 时必须提供非空 `routes`
- `project_code` 不可重复
- `rule_group` 必须是单位下已定义的真实规则组
- `rule_group` 不能引用保留字段 `template_selector`、`input_filename_routing`

行为：

- 匹配方式为文件名包含 `project_code`，且不区分大小写
- 命中后直接使用对应规则组处理
- 若同一文件名命中多个项目编码，程序会报错终止

## 15. `validation_rules`

```json
"validation_rules": {
  "required_fields": ["姓名", "实发工资"],
  "data_types": {
    "实发工资": "numeric",
    "发薪日期": "date"
  },
  "value_ranges": {
    "实发工资": {
      "min": 0.01,
      "max": 999999.99
    },
    "姓名": {
      "min_length": 2,
      "max_length": 30
    },
    "收入类型": {
      "allowed_values": ["01月收入", "年终奖", "补偿金"]
    }
  }
}
```

支持的顶层键：

- `required_fields`
- `data_types`
- `value_ranges`

不支持的旧键：

- `type_rules`
- `range_rules`

### `required_fields`

- 必须是字符串列表
- 在数据转换前执行

### `data_types`

允许的类型名包括：

- `numeric`
- `date`
- `datetime`
- `string`
- `str`
- `int`
- `integer`
- `float`
- `bool`
- `boolean`
- `list`
- `dict`

说明：

- 在数据转换后执行

### `value_ranges`

支持的子字段：

- `min`
- `max`
- `min_length`
- `max_length`
- `allowed_values`

约束：

- `min_length`、`max_length` 必须是 `>= 0` 的整数
- `allowed_values` 必须是列表
- `min` 和 `max` 若同时存在，必须同为数值边界或同为日期边界
- 在数据转换后执行

## 16. 运行时补充说明

### 零工资过滤

- 程序当前会固定按输入数据中的 `实发工资` 列做零值过滤
- 若输入中不存在该列，会直接报错

### 输出文件命名

默认模板：

```text
{unit_name}_{template_name}_{count}人_金额{amount:.2f}元{ext}
```

可用变量：

- `{unit_name}`
- `{month}`
- `{template_name}`
- `{count}`
- `{amount}`
- `{ext}`

若自定义模板中未显式使用 `{ext}`，程序会自动补上模板扩展名。

### 批量合并模式

- 只读取目标目录第一层中的 `.xlsx`、`.xls`
- 优先解析默认命名格式；若不完全匹配，会尝试根据配置中的单位名和模板名回推
- 文件按修改时间升序处理；若时间相同，按文件名升序
- 文件名中若提供了人数和金额，程序会与数据重算结果做一致性校验
